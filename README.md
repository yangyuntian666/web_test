## 项目概述

杨云天的AI小站是一个集技术博客、个人作品展示和互动游戏于一体的综合性个人网站。项目旨在展示AI技术应用、前端开发能力和游戏设计思维。

## 核心功能详解

### 主页设计
- **响应式布局**：使用CSS Grid和Flexbox实现自适应设计
- **视觉层次**：清晰的信息架构和导航路径
- **加载性能**：优化的资源加载策略

### 游戏中心
#### 坦克大战游戏特性
- **双平台支持**：同时适配PC和移动设备
- **渐进式难度**：随着关卡提升敌人AI智能
- **积分系统**：记录玩家成就和进度
- **生命机制**：三重生命保护机制

## 游戏机制

### 基础规则
- 玩家控制坦克击败所有敌方单位
- 每消灭一个敌人获得相应积分
- 敌人会从地图边缘持续生成
- 玩家有3次生命机会

### 游戏流程
1. 开始新游戏
2. 消灭当前关卡所有敌人
3. 进入下一关卡（难度提升）
4. 游戏结束条件：玩家生命耗尽或完成所有关卡

### 控制方案

#### PC端操作
| 操作 | 键位 |
|------|------|
| 向上移动 | W |
| 向左移动 | A |
| 向下移动 | S |
| 向右移动 | D |
| 射击 | 空格键 |

#### 移动端操作
- **虚拟摇杆区**：屏幕底部中央位置
  - 上/下/左/右方向按钮（55×55px）
- **独立操作区**：屏幕右下角
  - 射击按钮（60×60px）
- **辅助功能按钮**：
  - 全屏切换按钮
  - 游戏暂停按钮

## 移动端优化

### UI/UX设计原则
- **拇指友好**：主要操作区域位于屏幕下半部分
- **防误触设计**：方向键与射击键物理分离
- **视觉反馈**：按键按下时有明显状态变化
- **安全区域**：避开刘海屏和圆角区域

### 技术实现
- **触控事件处理**：使用Touch API监听触摸动作
- **响应式尺寸**：基于视口单位(vw/vh)进行布局
- **全屏API**：自动请求全屏模式提升沉浸感
- **性能优化**：节流和防抖确保流畅操作

### 适配策略
- **设备检测**：自动识别移动设备并启用触控模式
- **屏幕比例**：保持50vh的游戏画面高度
- **字体缩放**：使用rem单位确保文本可读性
- **图片优化**：根据设备像素比加载合适资源

## 技术实现

### 前端架构
- **HTML5**：语义化标签和Canvas元素
- **CSS3**：Flexbox布局、动画和过渡效果
- **JavaScript**：ES6+语法，模块化设计

### 关键技术点
- **Canvas渲染**：高效绘制游戏场景和角色
- **游戏循环**：requestAnimationFrame实现平滑动画
- **碰撞检测**：精确的矩形碰撞算法
- **状态管理**：游戏状态机控制不同场景

### 代码组织
- 模块化JavaScript代码
- CSS BEM命名规范
- 统一的错误处理机制
- 详细的代码注释
